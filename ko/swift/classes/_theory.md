Swift는 객체 지향 프로그래밍 언어로, 객체라고 불리는 프로그래밍 구조를 조작합니다.
객체는 데이터와 함수를 모두 포함하는 하나의 데이터 구조로 생각할 수 있습니다. 객체의 함수는 메서드라고 합니다.
예를 들어, 다음을 호출하면:
```swift
dictName.removeValue(forKey: "keyName")
```
Swift는 `dictName`에 `removeValue()` 메서드가 있는지 확인하고(모든 딕셔너리에 있습니다), 해당 메서드를 찾으면 실행합니다.

---

_구조체_와 _클래스_는 프로그램 코드의 기본 구성 요소가 되는 범용적이고 유연한 구조입니다.
기본적인 클래스|구조체는 `class` 또는 `struct` 키워드와 이름으로만 구성됩니다. 예를 들어:
```swift
class ClassName {
    // class definition
}
struct ClassName {
    // structure definition
}
```

---

클래스를 정의하는 것만으로는 객체가 생성되지 않습니다.
객체를 생성하려면 클래스의 __인스턴스__를 만들어야 합니다

---

클래스가 자체 함수를 가지고 있을 때, 그 함수들을 __메서드__라고 합니다.

---

Swift에서 구조체와 클래스는 많은 공통점을 가지고 있습니다. 둘 다 다음을 할 수 있습니다:
- 값을 저장하기 위한 프로퍼티 정의
- 기능을 제공하기 위한 메서드 정의
- 서브스크립트 문법을 사용하여 값에 접근하기 위한 서브스크립트 정의
- 초기 상태를 설정하기 위한 이니셜라이저 정의
- 기본 구현을 넘어 기능을 확장하기 위한 확장
- 특정 종류의 표준 기능을 제공하기 위한 프로토콜 준수

하지만 클래스는 구조체에는 없는 추가 기능을 가지고 있습니다:
- 상속을 통해 한 클래스가 다른 클래스의 특성을 상속받을 수 있습니다
- 타입 캐스팅을 통해 런타임에 클래스 인스턴스의 타입을 확인하고 해석할 수 있습니다
- 디이니셜라이저를 통해 클래스 인스턴스가 할당한 리소스를 해제할 수 있습니다
- 참조 카운팅을 통해 클래스 인스턴스에 대한 둘 이상의 참조가 가능합니다

---

_점 문법_을 사용하여 인스턴스의 프로퍼티에 접근할 수 있습니다.
점 문법에서는 인스턴스 이름 바로 뒤에 마침표 `.`로 구분하여 공백 없이 프로퍼티 이름을 작성합니다:
```swift
someInstance.someProperty
```
같은 문법을 사용하여 프로퍼티의 값을 업데이트할 수도 있습니다

---

구조체의 장점 중 하나는 자동으로 생성되는 멤버별 이니셜라이저가 있다는 것입니다. 이를 사용하여 새 구조체 인스턴스의 멤버 프로퍼티를 초기화할 수 있습니다.
새 인스턴스의 프로퍼티 초기값은 이름으로 멤버별 이니셜라이저에 전달할 수 있습니다
