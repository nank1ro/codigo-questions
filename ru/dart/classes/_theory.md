**Класс** — это шаблон для создания объектов. В Dart класс определяется с помощью ключевого слова `class`, после которого указывается имя класса и пара фигурных скобок:

```dart
class Animal {
  // поля и методы
}
```

По соглашению имена классов используют нотацию **PascalCase** (каждое слово начинается с заглавной буквы).

---

Класс может иметь **переменные экземпляра** (также называемые полями), которые хранят данные каждого объекта. Они объявляются внутри тела класса:

```dart
class Animal {
  String name;
  int age;
}
```

Каждый объект, созданный из этого класса, будет иметь собственные поля `name` и `age`.

---

**Конструктор** — это специальный метод, который вызывается при создании (инстанцировании) объекта класса. Конструктор имеет то же имя, что и класс:

```dart
class Animal {
  String name;

  Animal(String name) {
    this.name = name;
  }
}
```

Объект создаётся с помощью ключевого слова `new` (необязательного в Dart) или непосредственно по имени класса:

```dart
var dog = Animal('Rex');
```

---

Dart предоставляет сокращённый синтаксис конструкторов, который автоматически присваивает параметры переменным экземпляра. Вместо:

```dart
class Point {
  int x;
  int y;

  Point(int x, int y) {
    this.x = x;
    this.y = y;
  }
}
```

Можно написать:

```dart
class Point {
  int x;
  int y;

  Point(this.x, this.y);
}
```

Эта более короткая форма называется **инициализирующими формальными параметрами**.

---

**Метод** — это функция, определённая внутри класса. Методы описывают поведение объекта:

```dart
class Animal {
  String name;

  Animal(this.name);

  void speak() {
    print('$name издаёт звук.');
  }
}
```

Метод вызывается на объекте с помощью точечной нотации: `dog.speak()`.

---

`this` ссылается на **текущий экземпляр** класса. Он используется для различия между переменными экземпляра и параметрами с одинаковым именем:

```dart
class Circle {
  double radius;

  Circle(double radius) {
    this.radius = radius;
  }
}
```

Здесь `this.radius` ссылается на поле, а `radius` (без `this`) — на параметр конструктора.

---

Dart поддерживает **именованные конструкторы**, позволяющие определить дополнительные способы создания объекта. Именованные конструкторы записываются как `ИмяКласса.имяКонструктора`:

```dart
class Point {
  double x;
  double y;

  Point(this.x, this.y);

  Point.origin()
      : x = 0,
        y = 0;
}
```

Объект в начале координат создаётся следующим образом: `var p = Point.origin();`

---

**Геттер** — это специальный метод, который читает вычисляемое или приватное значение и выглядит как обращение к свойству. Он определяется с помощью ключевого слова `get`:

```dart
class Circle {
  double radius;

  Circle(this.radius);

  double get area => 3.14159 * radius * radius;
}
```

К геттеру обращаются как к полю: `circle.area` (без скобок).

---

**Сеттер** — это специальный метод, позволяющий присвоить значение с одновременным выполнением логики валидации. Он определяется с помощью ключевого слова `set`:

```dart
class Temperature {
  double _celsius = 0;

  double get celsius => _celsius;

  set celsius(double value) {
    if (value < -273.15) throw ArgumentError('Too cold!');
    _celsius = value;
  }
}
```

Имя поля часто начинается с `_`, что означает его приватность.

---

**Наследование** позволяет классу (**подклассу**) расширить другой класс (**суперкласс**) и повторно использовать его поля и методы. Для этого используется ключевое слово `extends`:

```dart
class Animal {
  String name;
  Animal(this.name);

  void speak() {
    print('...');
  }
}

class Dog extends Animal {
  Dog(String name) : super(name);

  @override
  void speak() {
    print('Woof!');
  }
}
```

`Dog` наследует `name` от `Animal` и переопределяет метод `speak`.

---

Когда конструктору подкласса нужно вызвать конструктор суперкласса, используется ключевое слово `super` в **списке инициализаторов**:

```dart
class Vehicle {
  String brand;
  Vehicle(this.brand);
}

class Car extends Vehicle {
  int doors;
  Car(String brand, this.doors) : super(brand);
}
```

`super(brand)` передаёт аргумент `brand` конструктору класса `Vehicle`.

---

**Переопределение методов** позволяет подклассу предоставить собственную реализацию метода, уже существующего в суперклассе. Для этого используется аннотация `@override`:

```dart
class Shape {
  double area() => 0;
}

class Circle extends Shape {
  double radius;
  Circle(this.radius);

  @override
  double area() => 3.14159 * radius * radius;
}
```

Аннотация `@override` сообщает Dart (и другим разработчикам), что метод суперкласса намеренно заменяется.

---

**Абстрактный класс** — это класс, который нельзя инстанцировать напрямую. Он используется как основа, определяющая контракт — методы, которые должны быть реализованы подклассами. Абстрактные методы обозначаются отсутствием тела:

```dart
abstract class Shape {
  double area(); // абстрактный метод — без тела
}

class Circle extends Shape {
  double radius;
  Circle(this.radius);

  @override
  double area() => 3.14159 * radius * radius;
}
```

Попытка напрямую инстанцировать `Shape()` приводит к ошибке.

---

**Статические члены** принадлежат самому классу, а не конкретному экземпляру. Они объявляются с помощью ключевого слова `static` и доступны непосредственно через имя класса:

```dart
class MathHelper {
  static const double pi = 3.14159;

  static double circleArea(double r) => pi * r * r;
}

// доступ без создания объекта:
print(MathHelper.pi);
print(MathHelper.circleArea(5));
```

Статические поля и методы разделяются всеми экземплярами.

---

**Фабричный конструктор** использует ключевое слово `factory` и позволяет управлять созданием объектов — например, возвращать кэшированный экземпляр или подтип:

```dart
class Logger {
  static final Logger _instance = Logger._internal();

  factory Logger() => _instance;

  Logger._internal();
}
```

Каждый вызов `Logger()` возвращает один и тот же экземпляр (паттерн синглтон).
