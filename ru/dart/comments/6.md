# --description--

Помимо этих двух способов написания комментариев, Dart включает комментарии _документации_.

Комментарии _документации_ — это многострочные или однострочные комментарии, которые начинаются с `///` или `/**`. Использование `///` на последовательных строках имеет тот же эффект, что и многострочный комментарий документации.

Комментарии _документации_ очень полезны, так как позволяют генерировать документацию.
Вам стоит добавлять комментарии _документации_ в свой код, чтобы было понятно, что делает тот или иной блок кода.

В комментарии _документации_ анализатор игнорирует весь текст, если он не заключён в квадратные скобки.
С помощью квадратных скобок можно ссылаться на _классы_, _методы_, _поля_, _переменные_, _функции_ и _параметры_.

Вот пример:
```dart
/// A domesticated South American camelid (Lama glama).
///
/// Just like any other animal, llamas need to eat,
/// so don't forget to [feed] them some [Food].
class Llama {
  String? name;

  /// Feeds your llama [food].
  ///
  /// The typical llama eats one bale of hay per week.
  void feed(Food food) {
    // ...
  }

  /// Exercises your llama with an [activity] for
  /// [timeLimit] minutes.
  void exercise(Activity activity, int timeLimit) {
    // ...
  }
}
```

В сгенерированной документации `[feed]` становится ссылкой на документацию метода `feed`, а `[Food]` — ссылкой на документацию класса `Food`.
В то время как `[activity]` и `[timeLimit]` становятся ссылками на документацию `activity` и `timeLimit` соответственно.
