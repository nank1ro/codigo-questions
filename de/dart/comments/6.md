# --description--

Zusatzlich zu diesen beiden Arten, Kommentare zu schreiben, umfasst Dart _Dokumentationskommentare_.

_Dokumentationskommentare_ sind mehrzeilige oder einzeilige Kommentare, die mit `///` oder `/**` beginnen. Die Verwendung von `///` auf aufeinanderfolgenden Zeilen hat denselben Effekt wie ein mehrzeiliger Dokumentationskommentar.

_Dokumentationskommentare_ sind sehr nutzlich, da sie die Generierung von Dokumentation ermoglichen.
Sie mochten _Dokumentationskommentare_ zu Ihrem Code hinzufugen, um klarzustellen, was ein bestimmter Codeblock tut.

Innerhalb eines _Dokumentationskommentars_ ignoriert der Analyser allen Text, es sei denn, er ist in eckigen Klammern eingeschlossen.
Mit eckigen Klammern kann man auf _Klassen_, _Methoden_, _Felder_, _Variablen_, _Funktionen_ und _Parameter_ verweisen.

Hier ist ein Beispiel:
```dart
/// A domesticated South American camelid (Lama glama).
///
/// Just like any other animal, llamas need to eat,
/// so don't forget to [feed] them some [Food].
class Llama {
  String? name;

  /// Feeds your llama [food].
  ///
  /// The typical llama eats one bale of hay per week.
  void feed(Food food) {
    // ...
  }

  /// Exercises your llama with an [activity] for
  /// [timeLimit] minutes.
  void exercise(Activity activity, int timeLimit) {
    // ...
  }
}
```

In der generierten Dokumentation wird `[feed]` zu einem Link zur Dokumentation der `feed`-Methode und `[Food]` zu einem Link zur Dokumentation der `Food`-Klasse.
Wahrend `[activity]` und `[timeLimit]` zu einem Link zur Dokumentation von `activity` bzw. `timeLimit` werden.
