# --description--

कमेंट्स लिखने के इन दो तरीकों के अलावा, Dart में _डॉक्यूमेंटेशन कमेंट्स_ भी शामिल हैं।

_डॉक्यूमेंटेशन कमेंट्स_ मल्टी-लाइन या सिंगल-लाइन कमेंट्स होते हैं जो `///` या `/**` से शुरू होते हैं। लगातार लाइनों पर `///` का उपयोग मल्टी-लाइन डॉक्यूमेंटेशन कमेंट के समान प्रभाव रखता है।

_डॉक्यूमेंटेशन कमेंट्स_ बहुत उपयोगी हैं क्योंकि वे डॉक्यूमेंटेशन उत्पन्न करने की अनुमति देते हैं।
आप अपने कोड में _डॉक्यूमेंटेशन कमेंट्स_ जोड़ना चाहेंगे ताकि यह स्पष्ट हो सके कि कोड का कोई विशेष ब्लॉक क्या करता है।

_डॉक्यूमेंटेशन कमेंट_ के भीतर, एनालाइज़र सभी टेक्स्ट को अनदेखा करता है जब तक कि वह वर्ग कोष्ठकों में संलग्न न हो।
वर्ग कोष्ठकों का उपयोग करके _classes_, _methods_, _fields_, _variables_, _functions_ और _parameters_ को संदर्भित किया जा सकता है।

यहाँ एक उदाहरण है:
```dart
/// A domesticated South American camelid (Lama glama).
///
/// Just like any other animal, llamas need to eat,
/// so don't forget to [feed] them some [Food].
class Llama {
  String? name;

  /// Feeds your llama [food].
  ///
  /// The typical llama eats one bale of hay per week.
  void feed(Food food) {
    // ...
  }

  /// Exercises your llama with an [activity] for
  /// [timeLimit] minutes.
  void exercise(Activity activity, int timeLimit) {
    // ...
  }
}
```

उत्पन्न डॉक्यूमेंटेशन में, `[feed]` `feed` मेथड के डॉक्यूमेंटेशन का लिंक बन जाता है, और `[Food]` `Food` क्लास के डॉक्यूमेंटेशन का लिंक बन जाता है।
जबकि `[activity]` और `[timeLimit]` क्रमशः `activity` और `timeLimit` के डॉक्यूमेंटेशन का लिंक बन जाते हैं।
