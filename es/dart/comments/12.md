# --description--

Además de estas dos formas de escribir comentarios, Dart incluye los _comentarios de documentación_.

Los _comentarios de documentación_ son comentarios de una sola línea o de múltiples líneas que comienzan con `///` o `/**.` El uso de `///` en líneas consecutivas tiene el mismo efecto que un comentario de documentación de múltiples líneas.

Los _comentarios de documentación_ son muy útiles porque permiten generar documentación.
Querrás agregar _comentarios de documentación_ a tu código para dejar claro qué hace un bloque de código particular.

Dentro de un _comentario de documentación_, el analizador ignora todo el texto a menos que esté entre corchetes.
Usando corchetes se puede referenciar a _clases_, _métodos_, _campos_, _variables_, _funciones_ y _parámetros_.

Aquí tienes un ejemplo:
```dart
/// Un camélido sudamericano domesticado (Lama glama).
///
/// Al igual que cualquier otro animal, las llamas necesitan comer,
/// así que no olvides de [alimentar]las con alguna [Comida].
class Llama {
  String? nombre;

  /// Alimenta a tu llama con [comida].
  ///
  /// La llama típica come una paca de heno por semana.
  void alimentar(Comida comida) {
    // ...
  }

  /// Ejercita a tu llama con una [actividad] durante
  /// [limiteDeTiempo] minutos.
  void ejercicio(Actividad actividad, int limiteDeTiempo) {
    // ...
  }
}
```

En la documentación generada, `[alimentar]` se convierte en un enlace a la documentación del método `alimentar`, y `[Comida]` se convierte en un enlace a la documentación de la clase `Comida`.
Mientras que `[actividad]` y `[limiteDeTiempo]` se convierten en enlaces a la documentación de `actividad` y `limiteDeTiempo` respectivamente.
