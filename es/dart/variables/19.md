# --instructions--

¿Cuál es la diferencia entre una constante de __compile-time__ y de __runtime__?

# --answers--

- Las constantes de __compile-time__ contienen valores fijos conocidos antes de ejecutar el programa, mientras que las constantes de __runtime__ obtienen un valor fijo solo durante la ejecución del programa
- Las constantes de __compile-time__ contienen valores fijos conocidos después de ejecutar el programa, mientras que las constantes de __runtime__ obtienen un valor fijo antes de ejecutar el programa
- No hay diferencia entre los dos
